# 1.基本概念
## 1.1 Token
大模型处理文本的**最基本单位**
`1Token`≈`1.5-2`个**汉字**≈4个**英文字母**（0.75个**英文单词**）
## 1.2 Context
大模型每次处理任务时所接收到的**信息总和**，大模型的**临时记忆体**
`Context Window`：`Context`能容纳的最大`Token`数量
## 1.3 Prompt
大模型接收的具体问题或指令
- `User Prompt`:具体任务，用户输入
- `System Prompt`:人设和做事规则，后台配置
## 1.4 Tool
给大模型提供一套可以调用的**外部能力**，让大模型能够**感知外部环境**
各角色的职责：
- 大模型
	- 选择要调用的工具，并生成工具参数
	- 归纳总结**工具**的执行结果
- 工具：执行具体的操作，如：查询天气
- 平台：串联整个流程，**调用工具**
## 1.5 MCP(**M**odel **C**ontext **P**rotocol)
统一的工具接入标准
## 1.6 Agent
**自主规划**、**自主调用工具**，直至完成用户任务的系统

## 1.7 Agent Skill
给Agent看的**说明文档**  
渐进式纰漏机制：
- 元数据层：名称和描述，**始终加载**
- 指令层：`SKILL.md`中除了名称和描述之外的内容，**按需加载**
- 资源层：`reference`和`script`，**按需中的按需加载**
## 1.8 RAG(Retrieval Augmented Generation)
1. 定义：检索增强生成
2. 功能：先从资料库中**检索**相关内容，再基于这些内容来**生成**答案
3. 应用场景：知识助手、智能客服
4. 运行流程：
- **用户提问之前**：
	1. 分片：把文档切分为多个片段。按照字数/段落、章节、页码进行分片。
	2. 索引：通过`Embedding`将片段文本转换为**向量**。将**片段文本**和**片段向量**存入**向量数据库**中。
	![[RAG提问前流程.png]]
- **用户提问之后**：
	1. 召回：搜索与用户问题相关片段的过程。采用**向量相似度**的方法，具有**成本低**、**耗时短**、**准确率低**的特点。**初步筛选**。
	2. 重排：使用`cross-encoder`方法，具有**成本高**、**耗时长**、**准确率高**的特点。**精挑细选**。
	3. 生成：将**用户问题**和**筛选的片段**发给大模型，后者根据片段内容回答用户问题。
